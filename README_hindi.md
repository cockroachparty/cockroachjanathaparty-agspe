## दृष्टिकोण / Vision

एजीएसपी एक सार्वजनिक हित विश्लेषणात्मक प्लेटफॉर्म के रूप में डिज़ाइन किया गया है जो इन चीजों को मिलाता है:

- ओपन-सोर्स इंटेलिजेंस (OSINT)
- क्रॉस-स्रोत पुष्टिकरण
- पूर्वाग्रह-जागरूक स्कोरिंग सिस्टम
- सार्वजनिक वित्तीय विश्लेषण
- पारदर्शी प्रायिकता पूर्वानुमान

यह प्रोजेक्ट कोकरोच जनता पार्टी के व्यापक दर्शन को दर्शाता है:

- विकेंद्रीकृत सार्वजनिक निगरानी
- भ्रष्टाचार विरोधी पारदर्शिता
- ओपन नागरिक तकनीक
- और सुलभ जांच टूलिंग

एजीएसपी किसी भी कॉर्पोरेट या सरकारी संस्था के साथ अंदरूनी पहुँच, विशेष सूचना, या राजनीतिक संबद्धता का दावा नहीं करता है। सभी विश्लेषण सार्वजनिक रूप से उपलब्ध डेटासेट, नियामक filings, समाचार आर्काइव, और एल्गोरिदमिक अनुमान मॉडल से व्युत्पन्न हैं।

## विशेषताएं / Features

- **प्रत्यायी विश्लेषण डैशबोर्ड** — वित्तीय संकेतकों (35%), राजनीतिक सहयोग स्कोरिंग (35%), और पैटर्न पहचान (30%) को जोड़ने वाला हाइब्रिड भविष्यवाणी इंजन, जिससे विश्वास अंतराल के साथ संभावित व्यावसायिक गतिविधियों का अनुमान लगाया जाता है
- **मल्टी-स्रोत पुष्टि इंजन** — तीन-स्तरीय स्रोत भरोसा वेटिंग (Tier 1 = 0.9, Tier 2 = 0.7, Tier 3 = 0.3) जिसमें क्रॉस-सत्यापन नियम, विरोधाभास पहचान, और ग्रुप-पक्षपाती स्रोत सीमा शामिल है
- **वित्तीय बुद्धिमत्ता** — 10 सूचीबद्ध आदानी ग्रुप कंपनियों का दोहरा मुद्रा (INR/USD) प्रदर्शन, मार्केट कैप, ऋण-से-संपत्ति हिटमैप, P/E अनुपात, और सोर्टेबल टेबल्स
- **लॉबी और राजनीतिक ट्रैकर** — 5 देशों में वैश्विक लॉबी खर्च, चुनाव बंधन दान ट्रैकर, कानूनी फर्म एंगेजमेंट मैपिंग
- **बुद्धिमत्ता फीड** — स्रोत टियर बैज के साथ फिल्टर करने योग्य न्यूज़ फीड, पुष्टि स्कोर बार, पूर्वाग्रह पहचान चेतावनियां, और कीवर्ड सर्च
- **अधिग्रहण इतिहास** — 20 प्रमुख अधिग्रहनों का कैटलॉग जिसमें मूल्य, स्थिति ट्रैकिंग, सेक्टर फिल्टरिंग, और वर्ष-रेंज फिल्टरिंग है
- **पूर्वाग्रह पहचान** — ऑटोमैटिक फ्लैगिंग ग्रुप-पक्षपाती स्रोतों का, Tier 1 की पुष्टि के बिना Tier 3 दावों का, और Tier 1 स्रोतों के बीच विरोधाभास अलर्ट
- **दैनिक डेटा पाइपलाइन** — Vercel Cron Jobs हर दिन 6 AM UTC पर सार्वजनिक स्रोतों से ताज़ा डेटा प्राप्त करते हैं (Reuters RSS, BBC RSS, SEC EDGAR), पुष्टि और भविष्यवाणी पाइपलाइनों को चलाने, और डैशबोर्ड अपडेट करते हैं
- **ज़ero-Config Deployment** — एक क्लिक में Vercel पर डेप्लॉय; बिल्ट-इन मॉक डेटा के साथ तुरंत काम करता है — कोई API की आवश्यकता नहीं

```

cp .env.example .env

```

यह शुरू करता है:

## Vercel Deployment Details

### यह कैसे काम करता है

एजीएसपी एकल Next.js application के रूप में Vercel पर पूरी तरह से चलता है:

1. **Frontend** — React components जो Next.js द्वारा rendered होते हैं, API routes से data fetch कर रहे हैं
2. **API Routes** — `/api/*` पर Serverless functions जो in-memory cache से data serve करते हैं
3. **Data Pipeline** — Vercel Cron daily `/api/cron/refresh` trigger करती है, जो:
   - Reuters RSS, BBC RSS, The Hindu RSS से latest news fetch करती है
   - SEC EDGAR public API से regulatory filings fetch करती है
   - सभी articles को Multi-Source Validation Engine से गुज़ारती है
   - Hybrid Prediction Engine का उपयोग करके नई predictions generate करती है
   - In-memory cache को validated, scored data के साथ update करती है
4. **Graceful Fallback** — यदि external sources unavailable हैं, तो सभी data comprehensive mock dataset पर fallback जाता है

### Cron Schedule

| Schedule | Endpoint | Description |
|----------|----------|-------------|
| Daily at 6 AM UTC | `/api/cron/refresh` | Fresh data fetch, validation & prediction pipelines run |

Cron `vercel.json` में configured है और deployment पर automatically activate होती है।

### Optional Environment Variables

| Variable | Purpose |
|----------|---------|
| `CRON_SECRET` | If set, `/api/cron/refresh` endpoint verifies `Authorization: Bearer <CRON_SECRET>` header |
| `NEWS_API_KEYS` | Enhanced news fetching के लिए comma-separated News API keys |

ये **पूरी तरह से optional** हैं — app के बिना भी काम करती है।

### Vercel Configuration

`vercel.json` file यह config करती है:

```json
{
  "crons": [{
    "path": "/api/cron/refresh",
    "schedule": "0 6 * * *"
  }],
  "functions": {
    "src/app/api/cron/refresh/route.ts": {
      "maxDuration": 60
    }
  },
  "headers": [{
    "source": "/api/(.*)",
    "headers": [
      { "key": "Cache-Control", "value": "s-maxage=3600, stale-while-revalidate=86400" }
    ]
  }]
}
```

- Cron daily 6 AM UTC पर run होती है
- Refresh function 60-second timeout रखती है (Vercel Hobby plan के लिए max)
- API responses 1 hour के लिए cached होते हैं 24-hour stale-while-revalidate के साथ

## Validation Engine

Multi-Source Verification Algorithm एजीएसपी की reliability का core है:

### Source Trust Tiers

| Tier | Weight | Examples | Behavior |
|------|--------|----------|----------|
| Tier 1 | 0.9 | Reuters, AP, BBC, Bloomberg, Financial Times, OCCRP | Highest trust; Tier 3 claims को corroborate कर सकता है |
| Tier 2 | 0.7 | The Hindu, Scroll.in, Business Standard | Independent Indian media; moderate trust |
| Tier 3 | 0.3 | NDTV, Times Now, Republic TV, Adani Group PR | Requires Tier 1 confirmation within 24hrs |

### Cross-Verification Rules

1. **Tier 3 claims** without Tier 1 corroboration within 24 hours get their validation score capped at 0.3 and are flagged "Requires Verification"
2. **Contradicting Tier 1 sources** manual review flag trigger करते हैं और confidence scoring pause करते हैं
3. **Pro-group-only sources** (all sources pro-Adani हैं) scores 0.2 पर capped होते हैं
4. **Single source penalty** — any claim backed by only one source 20% score reduction undergoes

### Output

| Field | Type | Description |
|-------|------|-------------|
| `validation_score` | Float (0.0–1.0) | Weighted confidence based on source tiers |
| `bias_risk_level` | String | "Low" / "Medium" / "High" |
| `flags` | Array | Specific validation flags raised |

## Data Pipeline

### Daily Refresh Flow

```
Vercel Cron (6 AM UTC)
    │
    ▼
/api/cron/refresh
    │
    ├── Fetch Reuters RSS → Parse articles
    ├── Fetch BBC RSS → Parse articles
    ├── Fetch The Hindu RSS → Parse articles
    ├── Fetch SEC EDGAR API → Parse filings
    │
    ▼
Validation Engine
    │
    ├── Apply tier weights (0.9 / 0.7 / 0.3)
    ├── Cross-verification checks
    ├── Contradiction detection
    ├── Pro-group bias capping
    └── Generate validation_score + bias_risk_level
    │
    ▼
Prediction Engine
    │
    ├── Calculate financial signals
    ├── Score political alignment
    ├── Match acquisition/policy patterns
    └── Generate predictions with confidence + timeline
    │
    ▼
Update Cache
    │
    ├── Store validated articles
    ├── Store fresh predictions
    ├── Store financial data
    └── Update lastRefresh timestamp
    │
    ▼
Dashboard Updated
```

### Graceful Degradation

If external data sources unreachable हैं:
1. Data fetcher error logs करती है और continue करती है
2. Store previous cache retain करती है (या mock data first run पर)
3. Predictions available data से generate होते हैं
4. Dashboard हमेशा data display करता है — कभी blank state नहीं दिखाता
5. `/api/health` endpoint data source को `"mock"` report करती है जब fallback data use होती है
